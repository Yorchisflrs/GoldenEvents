# Documentacion del Proyecto Golden Hour Events

## Objetivo del sistema

Crear una plataforma web para planificar eventos sociales en Puno, Peru, permitiendo que clientes armen eventos personalizados mediante servicios y cotizaciones.

## Roles

- Cliente: explora servicios, arma eventos personalizados y consulta cotizaciones.
- Organizador: crea y gestiona sus eventos.
- Proveedor: registra y gestiona servicios para eventos.
- Administrador: gestiona usuarios, servicios, cotizaciones y reportes.

## Modulos previstos

- Autenticacion y roles
- Catalogo publico de servicios
- Cotizaciones y detalles de cotizacion
- Reservas y pagos como modulos futuros
- Gestion de proveedores y servicios
- Administracion general
- Reportes basicos

## Fases del desarrollo

1. Parte 1: estructura base del proyecto y vistas minimas.
2. Parte 2: base de datos, relaciones, datos de prueba y conexion PDO.
3. Parte 3: logica funcional, roles, CRUD, reservas y validaciones.
4. Parte 4: diseno visual final, responsive y profesional.

## Recomendaciones de imagenes

- Usar formato WebP cuando sea posible.
- Comprimir imagenes antes de subirlas para mejorar rendimiento.
- Fondos hero o secciones principales: 1600x1000 px o 1920x1080 px recomendado.
- Peso recomendado por imagen de fondo: menor a 350KB.
- Tarjetas de servicios: 800x600 px recomendado.
- Evitar imagenes mayores a 2MB.
- Optimizar con TinyPNG, Squoosh o un conversor WebP antes de publicarlas.
- Mantener nombres de archivo descriptivos y sin caracteres especiales.
- Los fondos principales viven en `public/img/backgrounds/` y pueden migrarse de `.png` a `.webp` actualizando las rutas CSS.
